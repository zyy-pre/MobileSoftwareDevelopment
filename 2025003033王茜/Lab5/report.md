# Lab5 实验报告

## 一、应用内容说明
本次实验开发的 Art Space 应用，以世界经典名画为主题，实现了一个可循环浏览的数字艺术画廊。
应用包含3幅作品：
1. Mona Lisa（蒙娜丽莎）- Leonardo da Vinci (1503-1519)
2. Sunflowers（向日葵）- Vincent van Gogh (1888)
3. The Starry Night（星夜）- Vincent van Gogh (1889)

## 二、界面结构说明
应用界面采用模块化分区设计，整体使用垂直 Column 布局，分为三个核心功能区块：

1. **艺术作品墙**
   使用 Box + Image 组合实现，通过阴影和背景色营造画框效果，保证图片完整、等比例显示。

2. **作品信息区**
   使用 Column + Text 展示作品标题、作者与年份，使用不同字号和字重区分信息层级。

3. **控制按钮区**
   使用 Row 排列 Previous 和 Next 按钮，实现等宽分布与圆角样式，支持作品切换。

## 三、状态管理说明
应用使用 Jetpack Compose 状态管理实现界面动态更新：
1. 使用 remember + mutableStateOf 定义状态变量，记录当前显示的作品序号。
2. 图片、标题、作者信息全部根据状态变量动态获取，不使用硬编码。
3. 状态变化时，界面自动重组，实现无缝切换。

## 四、按钮逻辑说明
### Next 按钮逻辑
- 当前为第1幅 → 切换到第2幅
- 当前为第2幅 → 切换到第3幅
- 当前为第3幅 → 循环回到第1幅

### Previous 按钮逻辑
- 当前为第1幅 → 循环跳到第3幅
- 当前为第2幅 → 切换到第1幅
- 当前为第3幅 → 切换到第2幅

## 五、遇到的问题与解决过程
1. **图片无法显示**
   原因：图片文件名与代码不匹配。
   解决：统一命名为 artwork1.png、artwork2.png、artwork3.png 并放入 drawable 文件夹。

2. **按钮点击后界面不更新**
   原因：状态变量未正确使用 by 关键字。
   解决：使用 var currentArtwork by remember { mutableStateOf(1) } 实现状态驱动更新。

3. **图片变形、显示不全**
   原因：未设置图片缩放模式。
   解决：添加 contentScale = ContentScale.Fit 使图片按比例完整显示。

## 六、实验总结
本次实验综合运用 Jetpack Compose 基础组件、布局、状态管理和交互逻辑，完成了可循环切换的艺术画廊应用。通过声明式 UI 思想实现数据与界面的联动，理解了 Compose 状态驱动 UI 的核心机制，同时解决了图片加载、界面更新等实际问题，提升了 Android 开发实践能力。
