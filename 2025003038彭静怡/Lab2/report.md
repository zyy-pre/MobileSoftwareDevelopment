# Lab2 电子名片应用 - 实验报告

## 一、实验目标
使用 Jetpack Compose 实现一个简洁的电子名片界面，展示个人信息与联系方式，完成基础 UI 布局与组件使用。

---

## 二、布局思路
整体采用 **Column 垂直布局** 作为根容器，将界面分为上下两个功能区域，并统一背景色调：

1.  **上半部分（个人信息区）**
    -   嵌套 `Column` 垂直排列：**头像图标、姓名、职位**
    -   文字与头像均设置为水平居中对齐，突出个人身份信息
    -   使用 `Spacer` 控制元素间距，保证视觉舒适

2.  **下半部分（联系方式区）**
    -   嵌套 `Column` 包裹多行联系方式
    -   每行使用 `Row` 水平排列：**图标 + 文字**，清晰展示电话、邮箱、社交账号
    -   插入 `HorizontalDivider` 分割线，提升界面层次感与可读性

3.  **组件选型**
    全程使用 Compose 基础组件：`Column`、`Row`、`Icon`、`Text`、`Spacer`、`HorizontalDivider`，结构模块化、代码简洁，符合实验要求。

---

## 三、遇到的问题与解决方式

### 1. 组件/图标报错
- **问题**：组件导入路径错误，图标参数类型不匹配，导致 `Unresolved reference` 报错。
- **解决**：
  - 统一导入 `androidx.compose.material3.*` 组件包
  - 将图标参数类型从错误的 `Icon` 修正为 `ImageVector`，与系统图标类型匹配

### 2. `fillSize()` 标红报错
- **问题**：旧版 Compose API 已废弃，与当前项目版本不兼容。
- **解决**：将 `Modifier.fillSize()` 替换为官方推荐写法 `Modifier.fillMaxSize()`，功能不变且兼容新版。

### 3. 图片资源 `R.drawable` 报错
- **问题**：未准备本地头像图片，导致资源引用失败。
- **解决**：使用系统自带 Material 图标（如 `Icons.Default.Person`）替代本地图片，避免资源依赖问题，同时保持圆形头像样式。

### 4. 运行提示“进程已存在”
- **问题**：之前运行的 App 进程未正常关闭，引发安装冲突，无法重复启动。
- **解决**：
  - 在 Android Studio `Run` 面板中手动停止旧进程
  - 或重启模拟器后重新安装运行

成功实现电子名片界面，代码无编译错误，可正常运行与预览，界面布局清晰、配色协调，完整展示个人信息与联系方式，符合实验要求。
